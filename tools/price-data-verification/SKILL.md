# 股价数据验证铁律

> 2026-06-14 立：SK海力士 ₩2,150,000 被误读为 ₩215,000 这个低级错误不能再犯。

## 强制三步骤

### 步骤1：三源采集（最重要）

**每次采集股价，至少从 3 个独立数据源获取，全部写入报告。**

| 数据源 | 美股 | A股 | 港股/其他 |
|--------|:--:|:--:|:--:|
| **gtimg API（腾讯财经）** | ❌ | ✅🔴首选 | ✅ |
| Naver Finance API | ❌ | ❌ | ✅ 韩国 |
| Sina Finance | ✅ | ✅ | ✅ |
| Nasdaq API `/info` | ✅ | ❌ | ❌ |
| Investing.com | ✅ | ✅ | ✅ |
| Google Finance | ✅ | ✅ | ✅ |
| 同花顺/东方财富 | ❌ | ✅ | ❌ |
| ⚠️ Yahoo Finance | ✅ A股延迟2-3天 | ⚠️ 不推荐A股 | ✅ |

> 🔴 A股采集：gtimg > 同花顺/东方财富 > Sina > 其他。Yahoo Finance A股延迟2-3个交易日，仅作交叉参考。

三源格式（必须写入报告）：
```
📊 股价数据源交叉验证：
| 数据源 | 股价 |
|--------|------|
| Naver Finance | ₩2,150,000 |
| Investing.com | ₩2,150,000 |
| Google Finance | ₩2,150,000 |
→ 三源一致 ✅ 采用 ₩2,150,000
```

如果三源不一致：
```
📊 股价数据源交叉验证：
| 数据源 | 股价 | 判定 |
|--------|------|:--:|
| Nasdaq /info | $250.81 | ✅ 交易所直连 |
| Sina Finance | $247.30 | ⚠️ 延迟15分钟 |
| Google Finance | $250.81 | ✅ |
→ Sina延迟数据排除，Nasdaq与Google一致 ✅ 采用 $250.81
```

### 步骤2：PE 合理性检查（拿到价格后立刻做）

```python
PE = 股价 ÷ EPS（或一致预期EPS）

# 铁律：
if PE < 1:      → 🔴 价格大概率错了（差10倍/100倍）
if PE > 5000:   → 🔴 价格大概率错了
if 1 <= PE <= 5000: → 🟢 进入步骤2
```

**案例：SK海力士**
- Naver EPS(ttm) = ₩103,521，cnsEPS = ₩301,732
- 如果股价=₩215,000：PE = 215,000/301,732 = 0.71x → 🔴 触发警报
- 如果股价=₩2,150,000：PE = 2,150,000/301,732 = 7.12x → 🟢 与报告7.42x接近

### 步骤2：跨源交叉验证

- API 数据 + 用户截图/网页截图 → 两个来源一致才采信
- API 返回的数字如果"看起来需要除以10"，必须先做步骤1验证

### 步骤3：反向验算

从报告中取一个已知比率（如 Forward PE），反推股价：

```
报告Forward PE × EPS = 预期股价
7.42 × ₩28,300 = ₩209,986 ≈ ₩210,000 ✅
```

如果用 ₩21,000 × ₩28,300 = ₩594,300 ≠ ₩210,000 → ❌

## 韩国股票特别注意

- 韩国 API（Naver）返回的价格已经是真实韩元，**不要除以任何数**
- Investing.com 显示的数字就是真实韩元价格，**不要除以任何数**
- 韩国高价股（₩100,000+）PE通常在 5-50x 区间
- 如果算出的 PE < 1 或 > 5000，停止一切操作，先确认价格

### SK海力士血案（2026-06-14）

真实价格：**₩2,150,000**（215万韩元，Investing.com/Naver 三源一致）
报告误写：₩210,100（21万韩元，差10倍）

**错误链：**
1. 报告用만원单位写了 ₩210,100 → 我当成"21万韩元"
2. Naver API 返回 2,150,000 → 我自作聪明觉得"API习惯乘10"
3. 全程没做 PE 验算——如果做了：₩215,000 ÷ ₩301,732 = 0.71x ← 立刻发现价格错
4. 用户发了两次 Investing.com 截图写着 "2,150,000"，我还在圆"单位不同"

**教训：看到不符合预期的数字，第一反应是验算（PE），不是解释。**

0. **A股优先用gtimg** — 实时、免费、可靠。脚本：`scripts/batch_verify_prices.py`

## 内部一致性验算（v3.0 六项强制）

每份报告交付前，强制执行以下内部一致性检查：

| # | 检查项 | 公式 | 偏差 |
|---|--------|------|:--:|
| 1 | EPS × 股数 ≈ 净利 | `EPS × shares` | <1% |
| 2 | 股价 ÷ EPS ≈ PE | `price / eps` | <0.5% |
| 3 | 净利 < 营业利润 | `op profit > net profit` | 必须满足 |
| 4 | 概率加权 = Σ(p×t) | `p1×t1 + p2×t2 + p3×t3` | 精确 |
| 5 | PEG = PE ÷ 增速% | `pe / growth_rate` | <1% |
| 6 | 市值 = 股价 × 总股本 | `price × shares` | <1% |

执行方式：`python3 -c "..."` 逐项验算，grep找不出这些bug。

## 周期股/矿业股PE陷阱（🔴 v3.0新增）

对于矿业（金/铜/锂）、石油、钢铁、航运、养猪等周期行业，**低PE≠便宜**：

核心逻辑：当前EPS建立在不可持续的周期高点上。一旦商品价格回落，EPS暴跌→PE被动跳升。**2018年半导体Q2教训：PE 4-5x看似极便宜，随后EPS暴跌60%。**
